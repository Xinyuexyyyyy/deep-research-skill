# Example: Product Discovery To PRD

Prompt:

```text
我有个想法，想做成 PRD，帮我挖挖：一个给大学生用的 AI 简历成长系统。
```

Expected route:

```text
deep-research
-> Gate / Clarify
-> Wide Evidence And Visible Brainstorm
-> Adaptive Grillme
-> Narrowed Evidence Pass
-> PRD readiness
-> PRD or discovery_note
```

Expected behavior:

- Clarify target user, pain, and success standard.
- Brainstorm several possible product directions.
- Ask strong questions before writing specs.
- After narrowing, map evidence to the chosen direction.
- Preserve future vision while keeping MVP small.

PRD should include:

- target user
- pain
- value hypothesis
- MVP
- out of scope
- future roadmap
- vision / North Star
- acceptance criteria
- risks
- next step

Bad behavior:

- Writing a PRD immediately from the first prompt.
- Asking a long questionnaire all at once.
- Ignoring future roadmap because the MVP is lightweight.
