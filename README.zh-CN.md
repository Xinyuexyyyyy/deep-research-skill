# Deep Research Skill 中文说明

这是一个给 AI Agent 用的轻量调研工作流包。它的目标不是让 Agent 每次都写一份很长的报告，而是让它在面对模糊问题时，能一步步把问题问清、把方向展开、把证据打中，最后产出合适的结果。

一句话：**把“帮我调研一下”变成一条不会乱跑、不会早早下结论、也不会强行写 PRD 的工作链路。**

## 适合做什么

`deep-research` 适合这些场景：

- 市场调研、竞品分析
- 产品发现、MVP 定义、PRD 前调研
- 文献 / 论文方向的初步调研
- 模糊想法挖掘
- 用户痛点分析
- 开源工具和方法论扫描
- 综合决策，比如“这个方向值不值得做”

它不适合简单问答。比如“RAG 是什么？”这种问题应该直接回答，不应该启动完整调研流程。

## 这套东西的特色

普通的 deep research 很容易有几个问题：

- 一上来就搜索，没先问清“给谁用”和“为什么要做”。
- 只堆资料，不帮你看到可能方向。
- 过早收敛，错过更好的方案。
- 问问题像填表，一次扔十几个问题给用户。
- 证据还没打中，就开始写 PRD。
- 明明只是一个模糊想法，却被包装成很确定的产品方案。

这套 skill 的特点是：

- **先判轻重**：简单问题直接答，重问题才进正式调研。
- **先宽后窄**：先看多个可能方向，再让用户选择要深入哪条。
- **Brainstorm 可见**：不会把发散过程藏起来，用户能看到 3-5 个方向和放弃项。
- **Grillme 自适应**：只追问会改变结果的强问题，不固定 1-3 轮。
- **二次证据打中**：用户选定方向后，还要检查证据是否真的支撑这个方向。
- **PRD 有门槛**：不够清楚就降级为 `idea_note` / `discovery_note` / `decision_brief`，不硬写 PRD。
- **保留未来想象**：MVP 可以轻，但如果用户想做系统，PRD 必须保留 Roadmap 和 Vision。

## 核心流程

```text
Phase 0 Gate / Clarify
-> Phase 1 Wide Evidence + Visible Brainstorm
-> Phase 2 Adaptive Grillme
-> Phase 3 Narrowed Evidence Pass
-> Phase 4 PRD Readiness
-> PRD 或降级输出
```

## Phase 0：Gate / Clarify

目标：判断这是不是正式调研，以及先问清最关键的变量。

Agent 先判断：

- 这是简单解释 / 单点事实吗？如果是，直接答。
- 这是正式调研 / 产品发现 / PRD 前工作吗？如果是，进入流程。

正式调研前默认优先确认 3 个强问题：

1. **给谁用？** 目标用户、读者、客户或决策者是谁？
2. **痛点是什么？** 不解决这个问题会怎样？
3. **怎么才算成功？** 用户怎么判断这次调研有用？

这里的重点是“强问题”，不是“固定问卷”。如果上下文已经回答了，就不要重复问。如果用户说“你先看看有什么”，也可以带着假设进入 Phase 1，但要标注“未确认”。

Phase 0 的特色：

- 防止 Agent 一上来就盲搜。
- 防止做出来的东西读者不对、目标不对。
- 允许带假设前进，但不能假装已经确认。

## Phase 1：Wide Evidence + Visible Brainstorm

目标：先把可能性看宽，再收窄。

这一阶段至少看 3 条证据泳道：

| 证据泳道 | 解决的问题 | 典型来源 |
|---|---|---|
| 工具 / 开源项目 | 有什么可以直接借、改造、避开 | GitHub、产品文档、demo |
| 方法论 / 框架 | 成熟做法是什么，怎么判断好坏 | 指南、专家文章、论文、手册 |
| 竞品 / 产品 | 现有产品怎么包装、定价、差异化 | 官网、定价页、评论、案例 |
| 用户 / 社区痛点 | 真实用户怎么抱怨、绕路、卡住 | Reddit、HN、论坛、访谈 |
| 学术 / 数据 | 有没有研究证据、指标、理论支撑 | 论文、报告、数据集、白皮书 |

如果用户明确说“开源工具和方法论都要”，这两条必须并重，不能只给工具清单，也不能只讲原则。

Phase 1 要给用户看的不是资料堆，而是可选择的方向：

```text
我发现了这些可能方向：
- 方向 A：适合什么，优点是什么，缺点是什么
- 方向 B：适合什么，优点是什么，缺点是什么
- 方向 C：适合什么，优点是什么，缺点是什么

可借工具 / 项目：
- ...

可借方法论 / 框架：
- ...

暂时放弃：
- ...

我建议先深入 X，因为 ...
```

Phase 1 的特色：

- 用户能看到“宽”的过程，而不是只看到 Agent 私下做了选择。
- 每个方向都有利弊，方便人做判断。
- 明确写出放弃项，避免后面反复绕回旧方向。
- 这一阶段不写 PRD，不做最终方案。

## Phase 2：Adaptive Grillme

目标：在进入 PRD 或正式收敛前，把会导致跑偏的问题问出来。

`grillme` 的核心是：**一次只问一个重要问题**。

它把问题分成两类：

- **强问题**：会改变目标用户、范围、证据来源、MVP、验收标准或交付格式。
- **弱问题**：只影响语气、格式、篇幅、偏好。

只要还有强问题没解决，就继续问。只剩弱问题时，可以做默认假设继续。

这和普通“问卷式需求澄清”不一样。它不是固定问 1-3 轮，也不是一次扔十个问题，而是看每个答案是否会改变下一步产物。

## Phase 3：Narrowed Evidence Pass

目标：用户选了方向后，再确认证据真的支持这个方向。

这一阶段有两种做法：

1. 针对选定方向做一轮小范围二次检索。
2. 如果不需要再检索，就把 Phase 1 证据映射到当前决策。

映射格式：

| 已确认决策 | 支撑证据 | 证据强度 | 缺口 / 下一步验证 |
|---|---|---|---|
| ... | ... | 高 / 中 / 低 / 未验证 | ... |

如果证据支撑不了当前方向，就不能直接写 PRD，要继续取证或降级。

Phase 3 的特色：

- 防止“前面搜的是 A，后面写成 B”。
- 防止 grillme 后方向变了，但证据没有跟着更新。
- 让 PRD 的关键决策可追溯。

## Phase 4：PRD Readiness

目标：判断现在能不能写 PRD。

写 PRD 前必须满足：

- 目标用户清楚。
- 用户问题清楚。
- 不做什么清楚。
- MVP 切口清楚。
- 至少 3 个关键假设被列出。
- 每个关键假设有证据或验证动作。
- Narrowed Evidence Pass 已完成。
- 强问题已回答，或用户接受为未解决风险。
- 验收标准可衡量。
- 证据强度已标注。

如果不满足，就降级：

- `idea_note`：想法还太模糊。
- `discovery_note`：方向存在，但证据还不够。
- `decision_brief`：这是决策备忘，不是产品规格。
- `reader_brief`：只是资料理解，不该写 PRD。

## 安装方式

把两个 skill 文件夹复制到你的 Agent skill 目录。

Codex 常见路径：

```text
~/.codex/skills/deep-research/
~/.codex/skills/grillme/
```

然后开启一个新会话，让 Agent 重新加载 skill。

## 推荐测试

新会话里试这些 prompt：

```text
帮我调研一下 AI 代码工具市场，开源工具和方法论都要看。
```

预期：进入 `deep-research`，先 Gate / Clarify，后面 Phase 1 同时看工具和方法论。

```text
我有个想法，想做成 PRD，帮我挖挖：一个给大学生用的 AI 简历成长系统。
```

预期：先发散方向，再 grillme，再判断能不能写 PRD。

```text
找一下 LLM agent memory 这个方向的论文。
```

预期：先进 `deep-research`，再按需要内部走学术检索，不直接绕过主入口。

```text
简单解释一下 RAG 是什么。
```

预期：直接回答，不进入正式 deep research。
